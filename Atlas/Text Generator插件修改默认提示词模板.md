## Text Generator 有更高效的方法
核心是「模板切换」，不用反复去改默认设置。
### 核心方案：使用不同的模板指令，一键切换功能
你之前修改的 `Custom Instructions` 是全局默认模板，每次按 `Ctrl+J` 都会用这个规则生成。但你可以通过**临时指令**或**模板预设**，在不修改默认设置的情况下，随时切换不同功能。

### 方法1：在笔记里直接写指令（最省事，无需修改设置）
你只需要在要处理的内容上方，写一句指令，再选中指令+内容，按 `Ctrl+J` 即可。
举个例子：
```
请把下面的内容整理成3个要点：
（这里放你要处理的文字）
```
选中整段（包括指令和内容），按 `Ctrl+J`，插件就会按你写的指令生成要点，不会影响全局默认模板。
### 方法2：用「Slash Commands」保存常用模板（推荐，一次设置永久用）
插件的 `Slash-Suggest Options` 就是用来保存常用模板的，设置好后，输入 `/` 就能快速调用，不用每次都写长指令。

#### 操作步骤：
1.  打开 Text Generator 设置 → `Slash-Suggest Options`。
2.  点击「Add new」，给模板起个名字（比如 `摘要`）。
3.  在模板内容里写好固定指令，比如：
    ```
    请根据以下内容生成一段100字以内的摘要，并提取3-5个关键词：
    {{tg_selection}}
    ```
4.  保存模板。以后在笔记里输入 `/摘要`，插件就会自动插入这个指令，选中内容按 `Ctrl+J` 就能生成摘要了。
### 方法3：通过 Frontmatter 给单篇笔记指定模板
如果某篇笔记有固定的处理规则（比如所有读书笔记都要生成摘要），可以在笔记开头的 `---` 里加上：
```yaml
---
tg-instruction: 请为这篇读书笔记生成摘要和推荐标签
---
```
这样这篇笔记按 `Ctrl+J` 时，会优先用你写的指令，不用改全局设置。

---

### 总结
- 临时用一次：在笔记里直接写指令，选中+按 `Ctrl+J`。
- 经常用的功能：存成 Slash 模板，输入 `/` 一键调用。
- 特定笔记专用：写在 Frontmatter 里，不影响其他笔记。

这三种方法都不用反复修改 `Custom Instructions`，你可以根据场景选最顺手的用。
