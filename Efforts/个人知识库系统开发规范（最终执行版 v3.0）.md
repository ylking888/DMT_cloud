# 1. 项目目标
从零搭建一个基于 **Karpathy LLM Wiki 思路** 的个人知识库系统，实现：
- ✅ 三层架构严格隔离（ram / wiki / outputs）
- ✅ 来源溯源（SHA-256 哈希校验）
- ✅ 概念去重与别名对齐
- ✅ 强制结构化页面模板
- ✅ LLM 行为契约驱动（CLAUDE.md）
- ✅ qmd 索引支持
- ✅ 可持续演化与可审计
# 2. 三层架构模型（强制执行）
```
ram/        → 原始资料（人类拥有，只读）
wiki/       → 知识结构层（LLM 可读写）
outputs/    → 分析输出层（LLM 可读写）
```
## 2.1 权限规则

| 目录 | 人类 | LLM |
|------|------|------|
| ram/ | ✅ 读写 | ✅ 只读 ❌ 不可修改 |
| wiki/ | ✅ 读写 | ✅ 读写 |
| outputs/ | ✅ 读写 | ✅ 读写 |
⚠ LLM 严禁修改 `ram/` 目录。

# 6. scripts/lint.py 规范（9 项检查）
输出文件：
```
wiki/outputs/lint-YYYY-MM-DD.md
```
frontmatter 必须包含：
```yaml
graph-excluded: true
```
### 检查项
1. YAML frontmatter 合法性（含 type 与 date）
2. Broken wikiLinks
3. Index 一致性
4. Stubs 页面（正文 <100 字）
5. 近重复概念（Jaccard > 0.7）
6. SHA-256 完整性（SOURCE MODIFIED）
7. Stale 页面检测
8. 降重审查（URL 相似 + aliases 重叠）
9. Wikilink 格式规范（必须英文小写连字符）

# 8. INGEST 标准流程（11 步强制执行）
1. 读取 ram/ 中来源（只读）
2. 计算 SHA-256
3. 生成 slug（英文小写连字符）
4. 创建 wiki/sources/slug.md
5. 写入 ram_file + ram_sha256
6. 检查 2 年以上来源 → possibly_outdated: true
7. 概念名称对齐校验（slug + aliases 双重匹配）
8. 更新或创建 concept/entity 页面
9. 更新 source_count + confidence
10. 检查 QUESTIONS 是否可回答
11. 写入 wiki/log.md

## 8.1 个人写作流程（特殊）
- 不生成 Summary
- 不参与 source_count
- Evolution Log 格式：
```
YYYY-MM-DD 个人写作：[[slug]] 确立立场
```
# 9. QUERY 操作规范
步骤：
1. 执行 `qmd query`
2. 获取 top 5 相关页面
3. 每个核心结论必须溯源到 source 页面
4. 标注 confidence
5. 矛盾必须显式标注
输出类型自动匹配：

| 类型 | 格式 |
|------|------|
| 普通问题 | Markdown 正文 |
| 比较 | 表格 |
| 演示 | Markdown 幻灯片 |
| 趋势 | matplotlib 代码 |
| 清单 | bullet list |
# 10. REFLECT 四阶段
### Stage 0 — 反向检验  
主动寻找反证
### Stage 1 — 模式扫描  
批量读取 concepts/entities/synthesis
### Stage 2 — 深度合成  
生成 synthesis 页面
### Stage 3 — Gap Analysis  
识别孤立概念与盲区
完成后更新：
- overview.md
- index.md
- log.md
# 11. MERGE 规则
✅ 必须先询问用户  
✅ 主 slug 保留  
✅ 被合并页面改为 redirect  
✅ 更新所有 wikilinks  
✅ 记录 log  
跨语言合并：
- 主 slug 保留英文
- aliases 取并集
- Sources 去重合并
- My Position 需人工确认

# 12. Wikilink 使用规范（不可违反）
✅ 正确：
```
[[value-investing]]
[[attention-mechanism]]
[[warren-buffett]]
```
❌ 禁止：
```
[[价值投资]]
[[ValueInvesting]]
[[value_investing]]
```
中文名称处理方式：
- 写入 aliases
- Definition 第一行写：
  价值投资（Value Investing）

# 13. Wiki 语言规范
- Wiki 层统一中文写作
- slug 永远英文
- concept 页 title 使用中文主名称
- aliases 覆盖中英文

# 14. Confidence 更新规则

| 来源数 | 结果 |
|--------|------|
| 1 | low |
| 3 | medium |
| 5 且无重大矛盾 | 可提议 high |
| 用户确认 | 才可设 high |
⚠ personal-writing 不计入。

# 15. Source Integrity Rules
- lint 报 ▲ SOURCE MODIFIED → 必须重新 ingest
- 超过 2 年 → possibly_outdated: true
- 矛盾必须记录，不得覆盖
