## Claude Code 接入 DeepSeek V3（CC Switch 修复）

**问题描述**：CC Switch 写入 `~/.claude/settings.json` 时将 DeepSeek API Key 错误设为 `ANTHROPIC_AUTH_TOKEN` 字段，导致与官方 token 冲突，始终显示 Sonnet 4.6。

**根因**：`C:\Users\ylking\.claude\settings.json` 中字段名错误：
- 错误：`ANTHROPIC_AUTH_TOKEN: sk-bed713...`
- 正确：`ANTHROPIC_API_KEY: sk-bed713...`

**修复**：直接修改 `settings.json`，将 `ANTHROPIC_AUTH_TOKEN` 改为 `ANTHROPIC_API_KEY`。

**当前配置**（已生效）：
```json
{

    "ANTHROPIC_API_KEY": "sk-bed713f90590417d8dfa96a0d28bf0b6",
    "ANTHROPIC_BASE_URL": "https://api.deepseek.com/v1"
  },
  "claudeCodeSettings": {
    "model": "deepseek-chat"
  }
}
```
**最终正确配置**（已写入 `C:\Users\ylking\.claude\settings.json`）：
- Base URL：`https://api.deepseek.com/anthropic`（DeepSeek Anthropic 兼容接口，非 /v1）
- Model：`deepseek-v4-pro`（DeepSeek 官方 Anthropic 兼容模式支持的模型名）
- API Key：`sk-bed713f90590417d8dfa96a0d28bf0b6`

**踩坑记录**：
- `/v1` 是 OpenAI 兼容接口，Claude Code 发 Anthropic 格式请求会失败
- `/anthropic` 才是 DeepSeek 的 Anthropic 兼容接口
- Claude Code 内用 `/model deepseek-chat` 会报 not found，需用 `claudeCodeSettings.model` 字段指定
- CC Switch 将 Key 写为 `ANTHROPIC_AUTH_TOKEN` 而非 `ANTHROPIC_API_KEY`，导致冲突
